Static Website Hosting with High Availability

This project demonstrates how to host a static website on AWS with high availability and scalability.

Project Overview

The infrastructure is built using AWS CloudFormation, ensuring that resources are provisioned and managed as code.

Architecture

  • EC2 Auto Scaling Group behind an Application Load Balancer
  • Instances use Apache to serve static website files
  • Website content is pulled from an S3 bucket at launch
  • Elastic IP is used for persistent access

High Availability

Auto Scaling ensures that new instances are launched automatically in case of failures or increased traffic. The Load Balancer routes incoming traffic evenly across healthy instances.

Technology Stack

  • Amazon EC2
  • Elastic Load Balancer
  • S3 for storing website files
  • Auto Scaling Groups
  • Apache Web Server
  • CloudFormation for Infrastructure as Code

Conclusion

This project showcases a production-grade solution for hosting static websites using AWS services while ensuring availability, scalability, and automation.
